⚖️ Modos Quick vs Comprehensive

A cada análise você faz uma escolha: rápido e barato ou profundo e completo. Este módulo dissolve a confusão — mostra o que muda (e o que não muda), os modelos por trás, os custos lado a lado e uma regra simples para decidir.

Diagrama ilustrativo — os dois modos chegam aos mesmos documentos; o que muda é a profundidade da pesquisa.

Conteúdo detalhado

1

🔢 A diferença central

Existe um mito comum: o de que o modo Comprehensive "gera mais coisas". Não é verdade. A diferença é só uma — a profundidade da pesquisa. O Quick faz cerca de 9 buscas; o Comprehensive, cerca de 18. Os 15 documentos finais são exatamente os mesmos.

💡 A ideia central

Pense numa redação: os dois modos escrevem o mesmo número de páginas com a mesma estrutura. O Comprehensive simplesmente leu o dobro de fontes antes de escrever — então as páginas saem mais embasadas, com mais detalhe e nuance.

✓ O que MUDA entre os modos

  • ✓Número de buscas (9 vs 18)
  • ✓Profundidade e nuance dos documentos
  • ✓Tempo de execução
  • ✓Custo

✗ O que NÃO muda

  • ✗A quantidade de documentos (sempre 15)
  • ✗Os tipos de arquivo (PPTX, DOCX, PNG)
  • ✗A estrutura da pasta output/
  • ✗O modelo de síntese (sempre o mesmo Gemini)
2

🐇 Modo Quick

O Quick é o modo padrão — você não precisa nem digitar a flag. É rápido (2-3 minutos) e baratíssimo (~US$ 0,05). Pense nele como sua ferramenta de triagem.

⌨️ Rodando no modo Quick

# Estas duas linhas fazem a mesma coisa
python -m strategy_factory.main run "Stripe"
python -m strategy_factory.main run "Stripe" --mode quick

# ~9 buscas · 2-3 min · ~US$ 0,05

🎯 Quando usar o Quick

  • Triagem em lote — comparar várias empresas rapidamente
  • Primeira passada — ter uma visão geral antes de decidir
  • Testes e estudo — quando o custo precisa ser mínimo
  • Empresas que talvez nem vá aprofundar

💡 Dica — comece sempre aqui

Para a esmagadora maioria dos casos, o Quick já entrega um ótimo rascunho. Só suba para o Comprehensive quando o resultado for realmente para frente (cliente, diretoria).

3

🐢 Modo Comprehensive

O Comprehensive é a passada profunda. Leva de 5 a 10 minutos e custa cerca de US$ 0,50 — dez vezes mais que o Quick, mas ainda assim menos de um dólar. É o modo para a empresa que você vai realmente apresentar.

⌨️ Rodando no modo Comprehensive

# A flag é obrigatória aqui
python -m strategy_factory.main run "Stripe" --mode comprehensive

# Com contexto, o resultado fica ainda melhor
python -m strategy_factory.main run "Stripe" \
  --mode comprehensive \
  --context "B2B payments, fintech"

# ~18 buscas · 5-10 min · ~US$ 0,50

✓ Vale a pena quando

  • ✓O resultado vai para um cliente real
  • ✓Você vai apresentar para a diretoria
  • ✓A empresa é complexa ou pouco documentada
  • ✓Você quer o máximo de embasamento

✗ É exagero quando

  • ✗Você só está triando muitas empresas
  • ✗É um teste ou um estudo rápido
  • ✗Você ainda não decidiu se vai usar o resultado
  • ✗O tempo é mais crítico que a profundidade

💡 Dica — 10x mais caro ainda é centavos

"Dez vezes mais caro" soa muito, mas é a diferença entre 5 e 50 centavos de dólar. Quando o resultado importa, esse é o melhor investimento que você fará no dia.

4

🧠 Os modelos por trás

Por que o Comprehensive custa mais? Porque ele usa modelos de IA mais caros e potentes na pesquisa. Entender quais modelos entram em cada modo explica a conta.

🐇

Quick → só o sonar

Usa apenas o modelo sonar da Perplexity — barato e rápido. É o suficiente para uma boa visão geral.

🐢

Comprehensive → sonar + sonar-pro + deep-research

Combina sonar, sonar-pro e sonar-deep-research. Modelos mais fortes (e mais caros) que buscam fundo.

✍️

Síntese → sempre o Gemini

Nos dois modos, a escrita usa o mesmo modelo: gemini-2.5-flash. Por isso os documentos têm o mesmo formato — muda a matéria-prima, não o escritor.

🔑 Resumindo os modelos

  • Pesquisa (Quick): sonar
  • Pesquisa (Comprehensive): sonar + sonar-pro + sonar-deep-research
  • Síntese (sempre): gemini-2.5-flash

6

🧭 Como escolher

A estratégia campeã é simples: triar várias empresas no Quick e aprofundar só a escolhida no Comprehensive. E quando aprofundar, reaproveite a pesquisa que já tem.

🌳 Árvore de decisão

Você vai apresentar/entregar este resultado?
│
├─ NÃO  → 🐇 Quick (triagem, estudo, primeira passada)
│
└─ SIM  → 🐢 Comprehensive
          │
          └─ Já rodou Quick nesta empresa antes?
             │
             ├─ SIM → --mode comprehensive --skip-research?
             │        (avalie: pesquisa básica já serve?)
             │
             └─ NÃO → --mode comprehensive (pesquisa do zero)

⏱️ O fluxo recomendado, passo a passo

1

Trie no Quick

Rode run "Empresa" para cada candidata. Custa centavos no total.

2

Escolha a vencedora

Leia os rascunhos rápidos e decida qual empresa merece o aprofundamento.

3

Aprofunde só essa

Rode --mode comprehensive na escolhida. Use --skip-research se a pesquisa básica já bastar.

🎯 Resumindo

Filtre barato, invista caro só onde vale. Esse equilíbrio entre Quick e Comprehensive é o que mantém seus custos minúsculos sem abrir mão de qualidade quando importa.
